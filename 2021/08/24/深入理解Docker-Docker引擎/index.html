深入理解Docker-Docker引擎

随着云原生的日渐火爆，掌握docker对一名后端开发工程师来说是必不可少的技能。平时使用docker的时候也很少会去研究docker内部的实现原理，为了更好的掌握docker的使用，让我们一起揭开docker的神秘面纱吧。

Docker 引擎由如下主要的组件构成：

  • Docker 客户端（Docker Client）
  • Docker 守护进程（Docker daemon）
  • containerd
  • runc

总体架构图如下：

LXC

在最早的docker架构中，docker引擎使用的是LXC和Docker daemon

Docker daemon是单一的二进制文件主要是负责docker的客户端，api，以及容器运行时、镜像构建等

LXC 主要是对命名空间和控制组等基础工具的操作能力

LXC为Linux Container的简写。可以提供轻量级的虚拟化，以便隔离进程和资源，而且不需要提供指令解释机制以及全虚拟化的其他复杂性。相当于C++中的NameSpace。容器有效地将由单个操作系统管理的资源划分到孤立的组中，以更好地在孤立的组之间平衡有冲突的资源使用需求。

可以看出早期的docker是依赖LXC实现一些命名空间等基础操作的，而lxc是基于linux的容器虚拟化技术，很明显这对docker在今后的跨平台是很不利的，后来在docker0.9中开发了Libcontainer替代了LXC成为了默认的执行驱动

随着时间的发展，docker daemon这种整体的架构的笨重性日益突出，对于升级以及整个生态的建立都是不利的，所以应对的方法就是，拆！
